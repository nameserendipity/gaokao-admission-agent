# 高考志愿填报智能 Agent 项目进度记录

记录日期：2026-06-20
项目目录：`D:\高考志愿填报\projects`

## 当前目标

把现有高考志愿填报网页从纯前端 mock MVP，改造成可上线的智能 Agent MVP：

- 保留现有 `/input -> /preview -> /unlock -> /report` 流程。
- 报告生成迁移到服务端 API。
- 第一阶段覆盖浙江、山东。
- 后续真实数据进入 Supabase/Postgres。
- 接入 DeepSeek 做基于证据的自然语言分析和报告追问。
- 模型不得编造分数、位次、院校、专业或数据来源。

## 已完成改造

### 1. 服务端报告 API

已新增：

- `POST /api/reports/generate`
  - 输入考生画像。
  - 调用服务端推荐引擎生成报告。
  - 保存报告。
  - 返回 `reportId` 和报告内容。

- `GET /api/reports/[id]`
  - 根据报告 ID 读取完整报告。

- `GET /api/reports/[id]/chat`
  - 读取报告追问历史。

- `POST /api/reports/[id]/chat`
  - 基于当前报告证据回答用户追问。

相关文件：

- `src/app/api/reports/generate/route.ts`
- `src/app/api/reports/[id]/route.ts`
- `src/app/api/reports/[id]/chat/route.ts`

### 2. 数据库与仓储基础设施

已新增 Supabase/Postgres + Drizzle 基础设施：

- `src/lib/db/schema.ts`
- `src/lib/db/client.ts`
- `src/lib/db/repository.ts`
- `drizzle.config.ts`

设计表包括：

- `data_sources`
- `universities`
- `majors`
- `admission_records`
- `reports`
- `report_messages`

当前特性：

- 有数据库配置时走 Postgres。
- 没有数据库配置时自动回退到现有 mock 数据，便于本地直接跑通。
- 报告在无数据库时保存在服务端内存 Map 中。

### 3. 服务端推荐引擎

已新增：

- `src/lib/server/report-engine.ts`

当前能力：

- 校验用户画像。
- 根据省份读取录取数据。
- 无位次时按分数估算位次，并在报告里标记 `rankEstimated`。
- 根据选科、专业偏好、排除专业筛选录取记录。
- 按院校 + 专业聚合近年录取历史。
- 计算：
  - 位次差 `rankDiff`
  - 推荐类型：冲刺 / 稳妥 / 保底
  - 参考录取概率 `admissionChance`
  - 匹配分 `matchScore`
  - 风险等级 `riskLevel`
  - 捡漏候选 `isOpportunity`
- 每条推荐包含证据链：年份、最低分、最低位次、平均分、来源。

### 4. DeepSeek 适配

已新增：

- `src/lib/server/deepseek.ts`

当前策略：

- 如果配置 `DEEPSEEK_API_KEY`，调用 DeepSeek 生成报告解读和追问回答。
- 如果未配置 `DEEPSEEK_API_KEY`，自动使用规则化总结兜底，本地仍可运行。
- DeepSeek prompt 明确要求：只能基于结构化证据回答，不得编造分数、位次、院校、专业或来源。

相关环境变量：

```powershell
$env:DEEPSEEK_API_KEY="你的DeepSeek Key"
$env:DEEPSEEK_MODEL="deepseek-chat"
```

`DEEPSEEK_MODEL` 可不填，默认 `deepseek-chat`。

### 5. 前端页面改造

已改造：

- `src/app/preview/page.tsx`
  - 不再在浏览器直接调用 mock `generateReport`。
  - 改为调用 `POST /api/reports/generate`。
  - 保存 `reportId` 到 `sessionStorage`。
  - 显示服务端智能预览结论。

- `src/app/report/page.tsx`
  - 根据 `reportId` 调用 `GET /api/reports/[id]` 获取完整报告。
  - 展示冲刺、稳妥、保底推荐。
  - 展示数据证据链。
  - 新增“报告追问 Agent”模块。
  - 追问调用 `POST /api/reports/[id]/chat`。

保留：

- `/input` 表单流程。
- `/unlock` mock 解锁流程。
- 原有商业路径：预览 -> 解锁 -> 完整报告。

### 6. 数据导入脚本与命令

已新增：

- `scripts/seed-admissions.ts`

已新增 package scripts：

```json
"db:generate": "drizzle-kit generate",
"db:push": "drizzle-kit push",
"db:seed": "tsx scripts/seed-admissions.ts",
"agent:check": "tsx scripts/seed-admissions.ts --dry-run"
```

使用方式：

```powershell
$env:DATABASE_URL="你的Postgres连接串"
pnpm db:push
pnpm db:seed
```

当前 seed 脚本会把现有浙江、山东 mock 数据导入数据库，作为真实数据管线占位。后续需要替换/扩展为官方 CSV/Excel 清洗导入。

## 重要约定

- 不修复所谓中文乱码。用户已确认这是 PowerShell 显示问题，不是源码编码问题。
- 后续修改不要因为 PowerShell 输出乱码而重写原有中文文案。
- AI 只能解释数据，不能创造数据。
- 第一阶段只做浙江、山东。
- 数据更新采用定期采集入库，不做每次用户请求实时爬官网。
- 无数据库配置时必须保持 mock fallback，方便本地演示。
