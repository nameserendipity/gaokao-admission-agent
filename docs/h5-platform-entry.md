# H5 平台入口接入说明

## 当前阶段

云研志愿先以移动端 H5 主站承接流量：

- 主站：首页 `https://yunyanagent.com/`
- 平台承接页：`https://yunyanagent.com/share`
- 测评入口：`https://yunyanagent.com/input`

核心原则：先保证 H5 在微信、抖音、小红书等 App 内置浏览器可读、可填、可分享，再按平台资质逐步做小程序壳。

## 可挂载入口

### 微信

可用入口：

1. 微信聊天、微信群、朋友圈直接分享 H5 链接。
2. 公众号菜单配置 `/share` 或 `/input`。
3. 后续微信小程序使用 `web-view` 承载 H5。

注意事项：

- 小程序 `web-view` 需要配置业务域名。
- 如涉及支付、客服、订阅消息，建议优先走微信小程序能力。
- H5 分享标题、描述和图标已在 Next.js metadata 与 manifest 中配置。

### 抖音

可用入口：

1. 企业号主页、私信、允许的组件位引导到 H5。
2. 后续抖音小程序使用 `web-view` 或原生页面承载核心流程。

注意事项：

- 普通账号外链能力受平台规则限制。
- 小程序需要 AppID、主体资质、业务域名和平台审核。
- 抖音流量建议优先落到 `/share`，再进入 `/input`。

### 小红书

可用入口：

1. 主页链接或企业号组件入口。
2. 笔记内容引导搜索品牌词或进入主页链接。
3. 私信自动回复中引导到 H5。

注意事项：

- 小红书对外链跳转较克制，需结合账号类型和平台规则。
- 内容承接页必须短、可信、低跳失，因此优先使用 `/share`。

## 后续小程序化路线

推荐顺序：

1. H5 主站移动端体验稳定。
2. 微信小程序壳：登录、支付、客服、分享闭环。
3. 抖音小程序壳：短视频流量承接和表单转化。
4. 小红书企业号/小程序入口：内容种草与私域承接。

## 技术边界

- 各平台入口都调用同一套报告生成 API。
- 不为每个平台重写推荐算法。
- 小程序只做入口壳和平台能力适配。
- 任何平台都不得绕过选科硬规则、数据来源和免责声明。
